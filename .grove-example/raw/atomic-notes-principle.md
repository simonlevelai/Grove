# The Atomic Notes Principle

The atomic notes principle is a foundational concept in modern personal knowledge management. It states that each note should contain exactly one discrete idea — no more, no less.

## Why Atomicity Matters

A note containing a single idea can be linked, referenced, and reused in any context where that idea is relevant. A note containing three ideas can only be linked as a whole, dragging irrelevant content along with the relevant part.

Consider a note titled "Meeting with the design team" that contains discussion of a new colour palette, a decision to postpone the launch, and a reminder about a team dinner. This note is useful once — when recalling what happened in that meeting. An atomic note titled "We postponed the launch to improve onboarding flow" is useful every time you think about launch timing, onboarding, or the decision-making process.

## The Difficulty

Writing atomic notes is harder than writing long-form notes. It requires identifying the boundaries between ideas — where one concept ends and another begins. This analytical work is precisely what makes atomic notes valuable: the act of decomposition forces deeper understanding.

## Granularity

The right level of atomicity depends on the domain and the author's purpose. In academic research, an atomic note might capture a single finding or argument. In project management, it might capture a single decision. The test is: can this note be understood and used without reading another note?

## Composability

Atomic notes are composable. They can be arranged into sequences (arguments), clusters (topics), and hierarchies (from specific to general). This composability is what gives linked note systems their power — the same building blocks can construct different structures depending on the question being asked.
