# PARA and Building a Second Brain

Building a Second Brain (BASB) is a personal knowledge management methodology created by Tiago Forte. Its central premise is that human biological memory is unreliable for the volume and complexity of information modern knowledge workers encounter, and that an external "second brain" — a trusted digital system — can compensate [source: building-a-second-brain.md].

## The CODE Framework

BASB is structured around four steps:

**Capture** what resonates. Forte emphasises resonance over comprehensiveness: save what strikes you as interesting, surprising, or useful rather than trying to save everything [source: building-a-second-brain.md].

**Organise** for actionability using the PARA method. The question is not "what is this about?" but "what is this useful for?" [source: building-a-second-brain.md].

**Distil** to find the essence using [[progressive-summarisation]]. The goal is to make notes progressively more discoverable and usable over time [source: building-a-second-brain.md].

**Express** by turning accumulated knowledge into tangible output. Forte argues that knowledge has no value until it is expressed [source: building-a-second-brain.md].

## The PARA Method

PARA provides the organisational backbone of BASB. The acronym stands for four categories:

**Projects** are short-term efforts with a defined goal and deadline. **Areas** are ongoing responsibilities without an end date. **Resources** are topics of interest collected for future reference. **Archives** are inactive items from the other three categories [source: para-method.md].

The system is built on the principle that information should be organised by actionability, not by topic. A note about "machine learning" might live in Projects, Areas, or Resources depending on the author's current relationship to the topic [source: para-method.md].

PARA is deliberately tool-agnostic. The same four-folder structure works in a file system, a note-taking app, a cloud storage service, or an email client. Consistency across tools reduces cognitive overhead [source: para-method.md].

## Intermediate Packets

Forte emphasises "intermediate packets" — small, reusable units of work (summaries, outlines, drafts, diagrams) that can be recombined across different projects. This reduces the cost of starting from scratch and makes past effort compound [source: building-a-second-brain.md].

## Maintenance and Review

PARA requires periodic reviews to keep items in the correct category. Completed projects move to Archives. Resources that become relevant to an active project move to Projects. This movement keeps the system alive and prevents it from becoming a graveyard of stale information [source: para-method.md].

## Relationship to Other Approaches

Where the [[zettelkasten]] emphasises bottom-up emergent structure through linking, PARA provides a top-down organisational skeleton based on actionability. The two approaches are not mutually exclusive — many practitioners use PARA's folder structure alongside Zettelkasten-style linking within each category. [[Progressive-summarisation]] bridges the gap between raw capture and the polished [[atomic-notes-principle]] that a Zettelkasten requires [source: progressive-summarisation.md].
